STARTING THE APPLICATION
========================

Method 1: Double-Click Batch Script (Easiest)
  • Double-click: restart_app.bat
  • Handles everything automatically
  • Kills old process, starts fresh
  • Shows colored output with progress

Method 2: Command Line
  • Open Command Prompt or PowerShell
  • Navigate to app folder
  • Run: .\restart_app.bat
  • Or: python -m streamlit run app_streamlit_real_docs.py --server.port 8503

Expected Output:
  [1/4] Verifying Python installation...
  [OK] Python is available

  [2/4] Checking for existing process on port 8503...
  [INFO] No existing process found on port 8503

  [3/4] Verifying application files...
  [OK] Application file found: app_streamlit_real_docs.py

  [4/4] Starting RAG LOCALE application on port 8503...

  Access the application at: http://localhost:8503
  Press Ctrl+C to stop the server

WHAT GETS REMEMBERED
====================

1. Document Folder
   Storage: data/session_persistence/user_settings.json
   Duration: Until you change it

2. Loaded Documents
   Storage: data/session_persistence/documents_cache.json
   Duration: Until folder changes

3. Conversation History
   Storage: data/session_persistence/current_session.json
   Duration: Until you clear it

4. Document Topics
   Storage: data/topics_cache/topics_cache.json
   Duration: Until you clear it

5. Application Settings
   Storage: data/session_persistence/user_settings.json
   Duration: Persistent

TROUBLESHOOTING
===============

App won't start - Port 8503 in use:
  Solution 1: Restart batch script (automatic)
  Solution 2: netstat -ano | find "8503"
              taskkill /pid <PID> /f

Documents not remembered:
  Solution: Folder path changed or deleted
            Delete: data/session_persistence/user_settings.json
            Restart and select folder again

Cache seems broken:
  Solution: Delete data/session_persistence/documents_cache.json
            Restart and select folder again
            Wait for re-indexing

Slow on first load:
  Expected: 5-15 seconds is normal
  Reason: Indexing documents, extracting topics
  Next time: Will be instant from cache

ADVANCED: CLEAR CACHE
=====================

Option A: Keep folder, reload documents
  1. Delete: data/session_persistence/documents_cache.json
  2. Restart app
  3. Select folder again to re-cache

Option B: Complete reset
  1. Delete: entire data/session_persistence/ folder
  2. Restart app
  3. App treats as first time

Option C: Python script
  python -c "
  from session_persistence import SessionPersistence
  SessionPersistence.clear_session_cache()
  print('Cache cleared')
  "
